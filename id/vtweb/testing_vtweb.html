Pengujian VT-Web

Berikut ini daftar test case yang dapat dilakukan untuk membantu proses integrasi dengan Veritrans. Jika semua test case telah dilakukan, secara teknis website anda telah siap untuk GO LIVE dengan Veritans.

Test Case Hasil yang diharapkan
1 Meminta token ke Veritrans. Server Veritrans memberikan token_merchant dan token_browser.
2 Mengirim request redirection ke VT-Web. Berhasil diarahkan ke halaman pembayaran VT-Web.
3 Melakukan transaksi pembayaran dengan menggunakan nomor kartu kredit dummy Visa.
  • CC No: 4111 1111 1111 1111
  • CVV: 3 digit acak
  • Expiration Date: 2015-ke atas
 Transaksi sukses.
4 Melakukan transaksi pembayaran dengan menggunakan nomor kartu kredit dummy Mastercard.
  • CC No: 5555 5555 5555 4444
  • CVV: 3 digit acak
  • Expiration Date: 2015-ke atas
 Transaksi sukses.
5 Melakukan transaksi pembayaran dengan menggunakan nomor kartu kredit yang tidak valid.
  • CC No: 1111 1111 1111 1111
  • CVV : 3 digit acak
  • Expiration Date: 2015-above
 Transaksi gagal. Pelanggan akan melihat pesan "Please correct credit card information".
6 Pengujian 'finish redirection url'. Setelah proses pembayaran selesai, pelanggan akan otomatis diarahkan ke website merchant sesuai dengan alamat di finish_payment_return_url.
7 Pengujian HTTP(S) POST notification yang dikirim Veritrans setelah proses pembayaran. Server merchant akan mendapatkan notifikasi berupa HTTP(S) POST dari VT-Web yang berisi detail status transaksi. Sebelumnya, Silakan atur alamat yang akan menerima HTTP(S) POST tersebut di VT-Web Configuration Page.
8 Pengujian notifikasi email yg dikirim Veritrans ke pelanggan. Pelanggan akan mendapatkan notifikasi berupa email dari Veritrans yang berisi detail informasi dari sebuah transaksi pembayaran yang telah dilakukan.

Tips: Cara melakukan pengujian notification service di komputer lokal.

Saat Anda mengatur domain publik atau alamat IP publik pada halaman konfigurasi VT-Web untuk keperluan notifikasi VT-Web, alamat lokal seperti http://localhost/ atau http://127.0.0.1/ tidak bisa digunakan karena server VT-Web tidak bisa mengirim HTTP(S) POST request ke alamat lokal tersebut. Jadi, Anda perlu untuk men-deploy/meng-upload kode Anda ke web server/web hosting publik agar dapat melakukan testing HTTP(S) POST notification.

Namun demikian, ada alternatif lain agar kita dapat melakukan pengujian HTTP(S) POST notification di komputer lokal. Dengan Ngrok, kita akan mendapatkan alamat publik tanpa perlu men-deploy/meng-upload kode kita ke server publik. Syaratnya, komputer lokal kita harus terkoneksi dengan internet. Berikut ini langkah-langkahnya:

  1. Download Ngrok.zip di Ngrok.com.
  2. Ekstrak Ngrok.zip.

  3. Jika server lokal Anda suda berjalan di port 80 (http://127.0.0.1:80), buka terminal, lalu arahkan ke direktori hasil ekstraksi Ngrok.zip. Jalankan perintah ini: 

    $./ngrok 80

  4. Jika tidak ada masalah, terminal akan tampak seperti ini:

    Tunnel Status                 online
Version                       1.6/1.5
Forwarding                    http://3d311b98.ngrok.com -> 127.0.0.1:80
Forwarding                    https://3d311b98.ngrok.com -> 127.0.0.1:80
Web Interface                 127.0.0.1:4040
# Conn                        87
Avg Conn Time                 34.86ms

    Catatan: Alamat publik yang digenerate pada terminal Anda akan berbeda.

  5. Sekarang, localhost Anda sudah dapat diakses secara publik melalui internet. Dalam contoh di atas, setiap request ke http://3d311b98.ngrok.com bisa ditangkap di http://127.0.0.1:80.
  6. Setelah mempunyai alamat publik, Anda perlu mengatur Payment Notification URL di halaman konfigurasi VT-Web ke alamat publik tersebut, contohnya: http://3d311b98.ngrok.com/payment_notif_callback.
  7. Selesai. Pengujian notifikasi melalui HTTP(S) POST sudah bisa dilakukan.

  8. Tips tambahan: Ngrok merekam semua trafik HTTP request yang melalui Ngrok. Setelah Ngrok dijalankan, buka http://localhost:4040. Di halaman itu kita bisa melihat informasi detail semua request yang datang ke localhost kita.
