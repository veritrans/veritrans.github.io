Introduction VT-Direct

Target: Merchant understands about VT-Direct and how VT-Direct works

VT-Direct is one of Veritrans products that facilitates you to add payment feature for your online store. By using VT-Direct, your customer is able to process payment in their own website page without redirection.

VT-Direct will request a script from Veritrans server which will send the credit card information directly from the customer browser to Veritrans server without the data being saved in the merchant server.

How VT-Direct Operates

There are three participants during a payment process in VT-Direct: Merchant, customer, and Veritrans. The following is the explanation of how VT-Direct operates:

  1. Click Pay: Customer will fill the credit card details into the merchant website, and click Pay to complete payment.

  2. Send Card Data: When the customer click the pay button, a script called Veritrans.min.js is requested from Veritrans server and will combine the sensitive credit card information with the public-key. The combination of the data is then sent to Veritrans server to be exchanged with the onetime-token ("Token").

  3. Get Token: Veritrans will return the Token to the customer’s browser page. The token will be valid for 3 minutes during a standard credit card transaction and 10 minutes if a 3D Secure feature is utilised. Token will expire after exceeding the respective time.

    VT-Direct Process (Step 1 - 3)

  4. Send Token: Token is then sent to the merchant server, Do note that the merchant will only receive Token as a substitution for the credit card details. Merchant will not save or process any credit card information. Therefore, merchant does not have to encounter any PCI DSS security issue.

  5. Request Charge: Merchant will perform charge transaction with the Token, with addition of a few other information (Example: Customer details, product details, address, etc).

  6. Transaction Response Handling: Veritrans returns transaction status response, such as:

    • Capture
    • Settlement
