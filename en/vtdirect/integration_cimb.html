VT-Direct Integration (CIMB Clicks)

Every payment method has different integration steps available. The objective of this documentation is to assist you in integrating with VT-Direct CIMB Clicks payment method. For more inquiries, please contact us at support@veritrans.co.id or visit our support web page.



CIMB Clicks is one of the internet banking payment methods that is provided by CIMB Niaga Bank. This method allows the customer to perform a transaction by redirecting the customer to the CIMB Clicks payment page. For more information, please visit CIMB Niaga website.

To activate CIMB Clicks payment method (Automatically active on Sandbox mode), you have to contact a Veritrans sales representative.

Integration Steps VT-Direct CIMB Clicks

  1. Create Form
  2. Perform Charge Transaction
  3. Redirect
  4. Check Transaction Status

Unlike the other payment method (Credit Card and Mandiri ClickPay), CIMB Clicks payment process are not executed in your payment page. Customer will be redirected to CIMB Clicks page to complete the transaction.

  • Step 1Create Form
  • Step 2Perform Charge Transaction
  • Step 3Redirect
  • Step 4Check Transaction Status


Create Form

CIMB Niaga payment form will be made in the file form_vtdirect_cimb.php. Because most of the transaction process will be made in CIMB Clicks page, the form that we made will only show the steps of payment and the payment button of CIMB Clicks. When your customer clicks this payment button, charge transaction process will be carried on the next step.

The following is a sample from the file form_vtdirect_cimb.php: 

<html>
  <head>
    <title>VT-Direct CIMB Clicks Form Example</title>
  </head>
<body>
    <h1>CIMB Clicks Transaction</h1>
    <ol>
      <li>Make sure you have CIMB Clicks User ID and mPIN </li>
      <li>The balance is debited automatically according to the transaction value</li>
      <li>The minimum transaction value allowed is Rp. 10.000,00</li>
      <li>You have 1 hour to complete this transaction before the transaction expires automatically</li>
      <li>Make sure pop-up blocker is disabled in your browser</li>
    </ol>
    <form action="checkout_cimb.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
  </body>


Charge Transaction

To perform charge transaction using CIMB Clicks, the steps required are as follow:

  • payment_type parameter has to be set as cimb_clicks.

  • New JSON object has to be added, with the name “cimb_clicks”, and has the pairing name/value as followed:

    • "description" → transaction information

This step describes the charge transaction CIMB Clicks in the file checkout_cimb.php. To perform charge transaction, we will use Veritrans' PHP library. The following is an example of charge transaction implementation in the file checkout_cimb.php: 

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 20000,
);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );

// Data that will be sent for charge transaction request with CIMB Clicks.
$transaction_data = array(
  'payment_type' => 'cimb_clicks',
  'cimb_clicks' => array(
      'description' => "Description example",
    ),
  'transaction_details' => $transaction_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>


On the above example, we utilize the CIMB Clicks, transaction, and customer data as the charge transaction parameter. You can add additional data during the charge transaction, such as: Customer address, Delivery address, and item information. To add this data, you can check API Reference - API Methods


Redirect

Veritrans server will return a response for every charge transaction in JSON form. Veritrans PHP Library will convert JSON into a PHP object automatically. The sample JSON script that is returned by Veritrans is as followed: 

$result = array(
  "status_code" => "201",
  "status_message" => "Success, CIMB Clicks transaction is successful",
  "transaction_id" => "ff05337c-6c94-4f70-8e81-35acd89b688e",
  "order_id" => "201404141421",
  "payment_type" => "cimb_clicks",
  "transaction_time" => "2014-04-14 16:03:51",
  "transaction_status" => "pending",
  "redirect_url" => "https://api.sandbox.veritrans.co.id/cimb-clicks/request?id=ff05337c-6c94-4f70-8e81-35acd89b688e",
  "gross_amount" => "20000.00"
);


Two important information details in the JSON response are the status_code and redirect_url. If the value of status_code is 201, the charge transaction process is successfully saved in Veritrans server with pending status. That being so, you have to redirect your customer to redirect_url to finish the CIMB Clicks transaction. You can add the following PHP code at the last line of the file checkout.php to do so: 

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->status_code == "201")
{
  //success
  echo "Redirect CIMB Clicks. <br />";
  var_dump($result);
  header('Location: ' . $result->redirect_url);
} else {
  //error
  echo "There is an error on the transaction.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>


Information that is related to transaction value, order ID and transaction information will be displayed on the redirected URL page. On sandbox environment, the CIMB Clicks payment page is a dummy page. Customer simply has to click the payment button to complete the transaction process.

When the transaction is completed, customer can proceed back to your website. To do this, you have to fill in the callback URL configuration in MAP (Merchant Administration Portal). Error During Charge Transaction

Under a certain case, charge transaction on VT-Direct may fail. This will prevents you from getting to the redirect URL to CIMB Clicks page. Veritrans will return an error message if failure occurrs. The reason of failure can be inspected from the status code displayed on the error message. The status codes that generally occur are:

Error Code Explanation
400 Validation error Format or content of the transaction data is invalid; Example: Wrong variable type, invalid payment method.
401 Access Denied Invalid client or server key.
402 Access Denied Merchant does not have access to utilise a certain method of payment.
406 Duplicate Order ID Order ID has previously been used and paid.
410 Account Inactive Merchant account has been inactive for a long period of time.


The complete list of status code can be found at API Reference - Status Code.

When charge transaction process is completed, you can check the transaction status. The steps to examine transaction status will be discussed on the next steps.


Transaction Status

When charge transaction is successful, the transaction status is still pending. Customer has to complete the transaction in the CIMB Clicks page. When this process is completed, the transaction status will be changed into settlement or deny. You can check the transaction status change through MAP.
