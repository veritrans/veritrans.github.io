VT-Direct Integration (Mandiri Bill Payment)

Every payment method has different integration steps available. The objective of this documentation is to assist you in integrating with VT-Direct Mandiri Bill payment method. For more inquiries, please contact us at support@veritrans.co.id or visit our support web page.



Mandiri bill payment is a feature provided by Bank mandiri for their customer to easily make payments using a debit account. By using this, customers can pay bills anytime (24 hours)

To activate Mandiri Bill Payment method of payment (automatically active on Sandbox mode), you have to contact a Veritrans sales representative.

Integration VT-Direct Mandiri Bill Payment

  1. Create Form
  2. Perform Charge Transaction
  3. Display Bill Key
  4. Check Transaction Status

Unlike the credit card payment methods, Mandiri bill payment process is not executed on your payment page. Customer has to pay with Mandiri e-channel to complete the transaction.

  • Step 1Create Form
  • Step 2Charge Transaction
  • Step 3Display Bill key
  • Step 4Transaction Status


Create Form

Mandiri bill payment form will be made in the file form_vtdirect_mbill.html. Because most of the transaction process will not be made in the merchant's page, the form that we made will only show the steps of payment and the payment button of the Mandiri Bill Payment. When customer clicks this payment button, charge transaction process will be carried on the next step.

The following is a sample from the file form_vtdirect_mbill.html: 

<head>
  <title>Mandiri Bill Payment</title>
</head>
<body>
  <h1>Mandiri Bill Payment</h1>
  <b>I want to pay with Mandiri Bill payment</b>
  <form action="checkout_mbill.php" method="POST" id="payment-form">
      <button id="submit-button" type="submit">Pay</button>
    </form>
</body>


Charge Transaction

To perform charge transaction using Mandiri Bill payment, the following steps are required:

  • payment_type parameter has to be set as echannel.

  • New JSON object has to be added, with the name echannel, and has the pairing name/value as followed:

    • bill_info1 → Label information 1
    • bill_info2 → Value information 1

This step describes the Mandiri Bill Payment charge transaction in the file checkout_mbill.php. To perform charge transaction, we will use Veritrans PHP library. The following is an example of charge transaction implementation in the file checkout_mbill.php: 

<?php
include_once('Veritrans.php');

// TODO: change with your actual server_key that can be found on Merchant Administration Portal (MAP)
Veritrans_Config::$serverKey = "< Your Server Key >";
Veritrans_Config::$isProduction = false;

$transaction_details = array(
  'order_id'    => rand(),
  'gross_amount'  => 145000,
);

$item1_details = array(
    'id' => 'a1',
    'price' => 50000,
    'quantity' => 2,
    'name' => 'Apel'
    );

$item2_details = array(
    'id' => 'a2',
    'price' => 45000,
    'quantity' => 1,
    'name' => 'Jeruk'
    );

$items_details = array ($item1_details, $item2_details);

$customer_details = array(
    'first_name'    => "Andri",
    'last_name'     => "Litani",
    'email'         => "andri@litani.com",
    'phone'         => "081122334455"
    );


// Data that will be sent for charge transaction request with Mandiri Bill.
$transaction_data = array(
  'payment_type' => 'echannel',
  'echannel' => array(
      'bill_info1' => "Payment for",
      'bill_info2' => "debt"
    ),
  'transaction_details' => $transaction_details,
  'item_details'        => $items_details,
  'customer_details' => $customer_details
  );

$result = Veritrans_VtDirect::charge($transaction_data);
?>


On the above example, we utilize the mandiri bill payment data, transaction data, and customer data as the charge transaction parameter. You can add additional data during the charge transaction, such as: Customer address, delivery address, and item information. To add this data, you can check API Reference - API Methods.


Display Bill Key

Veritrans' server will return a response for every charge transaction in JSON form. Veritrans' PHP Library will convert JSON into a PHP object automatically. The sample JSON script that is returned by Veritrans is as followed: 

$result = array(
  "status_code" => "201",
  "status_message" => "Success, Mandiri Bill transaction is successful",
  "transaction_id" => "ff05337c-6c94-4f70-8e81-35acd89b688e",
  "order_id" => "201404141421",
  "payment_type" => "echannel",
  "transaction_time" => "2014-04-14 16:03:51",
  "transaction_status" => "pending",
  "bill_key" => "060811223344",
  "biller_code" => "70012",
  "gross_amount" => "145000.00"
);


Two important information details in the JSON response above are the status_code, biller_code and bill_key. If the value of status_code is 201, the charge transaction process is successfully saved in Veritrans server with pending status. Merchant is required to show the biller_code and bill_key to the customer so they can finish the transaction. You can add the following PHP code at the last line of the file checkout_mbill.php to do so: 

<?php
include_once('Veritrans.php');

...

$result = Veritrans_VtDirect::charge($transaction_data);

if($result->status_code == "201")
{
  //success
  echo "<b>Mandiri Bill payment</b> <br /><br />";
  echo "<b>Please complete payment using Mandiri e channel with this informations:</b> ";
  echo "<b>Company Code:</b> ".$result->biller_code;
  echo "<b>Bill Key:</b> ".$result->bill_key;
  echo "<br /><br />";
  echo "1. You have 24 hours to complete payment before this transaction expired <br />";
  echo "2. If you need help, please contact 021-22334456";

} else {
  //error
  echo "Have error on transaction.<br />";
  echo "<h3>Result:</h3>";
  var_dump($result);
}
?>


Up to this point, the transaction is stored in MAP with pending status. Transaction status will change to success when your customer completes the payment.

Info: Note:
To facilitate the payment process, we provide the payment simulator in sandbox Environment.


Error During Charge Transaction

Under a certain case, charge transaction on VT-Direct may fail. This will prevent you from getting the virtual account number. Veritrans will return an error message if failure occurrs. The reason of failure can be inspected from the status code displayed on the error message. The status codes that generally occur are:

Error Code Explanation
400 Validation error Format or content of the transaction data is invalid; Example: Wrong variable type, invalid payment method.
401 Access Denied Invalid client or server key.
402 Access Denied Merchant does not have access to utilise a certain method of payment.
406 Duplicate Order ID Order ID has previously been used and paid.
410 Account Inactive Merchant account has been inactive for a long period of time.


The complete list of status code can be found at API Reference - Status Code.

When charge transaction process is completed, you can check the transaction status. The steps to examine transaction status will be discussed on the next steps.


Transaction Status

When charge transaction is successful, the transaction status is still pending. Customer has to complete the transaction. When this process is completed, the transaction status will be changed into success. You can check the transaction status change through MAP.

HTTP(S) Response

Veritrans' server will return a response to your server when customer has complete the payment. The sample JSON script that is returned by Veritrans is as followed: 

{
    "status_code": "200",
    "status_message": "Veritrans payment notification",

    "payment_type": "echannel",
    "transaction_time": "2014-11-07 10:43:17",
    "transaction_status": "settlement",
    "bill_key" => "060811223344",
